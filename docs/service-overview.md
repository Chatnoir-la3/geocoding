# 地理空間MCP Server — サービス活用ガイド

## 1. MCP Serverとは何か

MCP（Model Context Protocol）は、AIエージェントが外部データソースやツールに標準化された方法でアクセスするためのプロトコル。地理空間MCP Serverは、国土交通省「不動産情報ライブラリ」APIが提供する**30種類の不動産・地理空間オープンデータ**をAIが自然言語で取得・分析できる環境を提供する。

```
ユーザー → 「渋谷駅周辺の災害リスクと地価を教えて」
    ↓
AI（Claude等） → MCP経由で複数APIを自動判断・並列取得
    ↓
構造化データ → 地価5,500,000円/㎡、洪水浸水0.5m未満、用途地域：商業地域...
```

**従来**: 不動産情報ライブラリのWebサイトで手動レイヤー切替・地図操作が必要
**MCP活用後**: 自然言語の1文で複数データを横断取得・分析

---

## 2. 取得可能な30種のデータ

| カテゴリ | データ種別 | 主な活用場面 |
|---------|-----------|------------|
| **価格情報** | 地価公示、地価調査、取引価格、鑑定評価、成約価格 | 不動産査定、投資判断、相場把握 |
| **都市計画** | 用途地域、防火地域、都市計画区域、地区計画、高度利用地区、都市計画道路、立地適正化 | 建築計画、開発可否判断、法規制確認 |
| **周辺施設** | 学区（小中学校）、幼稚園・保育園、医療機関、図書館、駅乗降客数 | 居住地選定、子育て環境評価、商圏分析 |
| **災害リスク** | 洪水、土砂災害、津波、高潮、地すべり、急傾斜地、災害危険区域、液状化、避難場所、災害履歴 | リスク評価、保険査定、BCP策定 |
| **地形・地盤** | 土地条件図、陰影起伏図、大規模盛土造成地 | 地盤調査、造成リスク評価 |

## 3. サービス展開シナリオ

### 3-1. 不動産仲介・売買支援

**課題**: 顧客への物件説明に複数サイトの情報を手作業で収集
**MCP活用**:

```
「この物件周辺の地価推移、用途地域、最寄り駅の乗降客数、
 学区情報、災害リスクをまとめて」
→ 1回の自然言語指示で物件調査レポートを自動生成
```

- **物件査定の根拠資料作成**: 地価公示・取引価格・成約価格の3データを組み合わせた査定根拠
- **重要事項説明の補助**: 用途地域、防火地域、災害リスク情報の自動収集
- **顧客向け物件レポート**: 周辺環境（学校、医療、利便性）の包括的な資料生成

**想定ユーザー**: 不動産仲介会社、不動産ポータルサイト

---

### 3-2. 不動産投資分析

**課題**: 投資判断に必要な複数指標の収集・比較に時間がかかる

**MCP活用**:
- **エリア比較分析**: 「渋谷区と港区の地価推移・取引価格・乗降客数を比較して」
- **再開発ポテンシャル評価**: 立地適正化計画 + 都市計画道路 + 地価データの複合分析
- **利回り・リスク評価**: 取引価格 × 災害リスク × 用途地域の総合スコアリング

**想定ユーザー**: 不動産投資家、アセットマネジメント会社、REIT運用者

---

### 3-3. 住まい探し・移住支援

**課題**: 引越し先の総合的な住環境評価が難しい

**MCP活用**:
```
「子供2人（小学生・保育園）で横浜市に引越し検討中。
 学区、保育園、医療機関、災害リスク、地価をまとめて」
→ 子育て環境スコアリング + 安全性評価 + コスト感を統合レポート
```

- **子育て世帯向け**: 学区 + 保育園 + 医療機関の半径圏内検索
- **災害リスク重視層向け**: 洪水・津波・液状化・土砂災害の複合リスク評価
- **移住希望者向け**: 地方自治体の立地適正化計画から「住みやすいエリア」を推薦

**想定ユーザー**: 住宅情報サービス、自治体の移住促進窓口

---

### 3-4. 金融・保険業務

**課題**: 融資審査や保険査定で個別にデータ収集するコストが大きい

**MCP活用**:
- **住宅ローン審査**: 担保物件の地価・用途地域・災害リスクの自動評価
- **火災保険査定**: 防火地域区分 + 洪水浸水深 + 液状化リスクの複合判定
- **企業向けBCP策定**: オフィス・工場の所在地に対する複合災害リスクレポート

**想定ユーザー**: 銀行、保険会社、損保

---

### 3-5. 自治体・公共サービス

**課題**: 住民からの問い合わせ対応に専門知識が必要

**MCP活用**:
- **窓口チャットボット**: 「この土地に家を建てられますか？」→ 用途地域 + 防火地域 + 災害リスクを即時回答
- **防災情報提供**: 住所入力で洪水・津波・土砂災害リスク + 最寄り避難場所を一括表示
- **都市計画の住民説明**: 立地適正化計画・地区計画の対象範囲を自然言語で説明

**想定ユーザー**: 市区町村の都市計画課、防災課、住民窓口

---

### 3-6. 建設・設計業務

**課題**: 設計前の敷地調査で複数の規制情報を個別に確認する必要がある

**MCP活用**:
- **敷地調査の自動化**: 用途地域 + 防火地域 + 高度地区 + 都市計画道路を一括取得
- **地盤リスク事前調査**: 土地条件図 + 大規模盛土 + 液状化の複合評価
- **施工計画の安全評価**: 災害リスク + 急傾斜地 + 地すべり防止区域の確認

**想定ユーザー**: 設計事務所、ゼネコン、ハウスメーカー

## 4. ビジネスモデルの方向性

### A. SaaS型 — 不動産AIアシスタント

```
月額課金 → チャットUI + レポート自動生成 + API利用枠
```

- 不動産仲介向け: 物件調査レポート自動生成SaaS
- 投資家向け: エリア分析・比較ダッシュボード
- 差別化要素: 30種のオープンデータを統合した分析は手動では非常に手間

### B. API型 — 地理空間データ統合API

```
従量課金 → 既存不動産サービスへのAPI組込み
```

- 不動産ポータル（SUUMO、HOMES等）への組込みAPI
- 既存CRMやSFAへのプラグイン
- MCPプロトコル準拠により、Claude/GPT等のAIエージェントから直接利用可能

### C. レポート型 — 調査レポート自動生成

```
件数課金 → 物件調査レポートPDF生成
```

- 重要事項説明の補助資料
- 投資判断レポート
- 災害リスク評価書
- 差別化: AIによる自然言語解説 + データの構造化

### D. 自治体向け — 住民サービスDX

```
年間ライセンス → 窓口チャットボット + 防災情報提供
```

- 都市計画・防災情報の住民向け対話型提供
- 移住促進ポータルへの組込み

## 5. 競合優位性と差別化ポイント

| 要素 | 従来のサービス | MCP活用サービス |
|------|-------------|---------------|
| データ取得 | サイト別に手動操作 | 自然言語で一括取得 |
| 分析 | 専門知識が必要 | AIが自動で解釈・要約 |
| クロス分析 | 複数サイト横断で困難 | 30種データの複合分析が容易 |
| コスト | 人件費（調査工数） | API利用料のみ |
| スケーラビリティ | 人的リソースに依存 | APIスケーリングで対応 |
| データソース | 国土交通省オープンデータ（信頼性高） | 同左 |

- **不動産業界の課題**: 約6割の企業が「AI活用方法の理解不足」（AI-Market調査）
- **トレンド**: AI活用がPOC段階から実運用段階へ移行中（PwC 2025 Emerging Trends）

## 8. まとめ

地理空間MCP Serverの最大の価値は、**30種の信頼性の高い公的オープンデータを、AIが自然言語で横断的に取得・分析できる**点にある。

**短期（〜6ヶ月）**: 不動産仲介向け物件調査AIアシスタント → 業務工数削減の実証
**中期（6ヶ月〜1年）**: 投資分析・金融向けSaaS化 → 収益モデル確立
**長期（1年〜）**: 自治体DX・他MCPとの統合プラットフォーム → エコシステム構築

国土交通省がMCPサーバーを公式に公開したことは、**不動産×AI市場の本格的な立ち上がりのシグナル**であり、この基盤の上にサービスを構築する先行者優位が期待できる。
